DCSync

DCSync is an Active Directory attack that allows an attacker to impersonate a Domain Controller and request password data through the Directory Replication Service (DRS). Instead of accessing the NTDS.dit database directly, the attacker abuses the normal replication process to retrieve NTLM hashes and Kerberos keys for any domain account.

How It Works

  1. The attacker gains an account with directory replication privileges.
  2. A rogue replication request is sent to the Domain Controller using the DRS Remote Protocol (DRSUAPI).
  3. The Domain Controller treats the attacker as a legitimate replication partner.
  4. Password hashes and Kerberos keys for the requested accounts are returned.

Requirements

  • An account with Replicating Directory Changes, Replicating Directory Changes All, and Replicating Directory Changes in Filtered Set permissions.
  • Network access to a Domain Controller.
  • Valid domain credentials.

Failure Conditions

  • The required replication permissions are not assigned.
  • Replication traffic is blocked or restricted.
  • The replication request is denied by the Domain Controller.

Usage Scenarios

  • Dumping credentials for any domain user, including Domain Administrators.
  • Obtaining NTLM hashes for Pass-the-Hash or offline password cracking.
  • Retrieving Kerberos keys for additional Kerberos-based attacks.
  • Post-exploitation after compromising a privileged account.

Important Notes

  • Does not require direct access to the NTDS.dit file.
  • Abuses the legitimate Active Directory replication mechanism.
  • By default, Domain Admins, Enterprise Admins, and Administrators possess the required replication rights.
  • One of the most powerful Active Directory attacks because it can retrieve credentials for any domain account.
