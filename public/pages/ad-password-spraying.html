Password Spraying

Password Spraying is a brute-force technique in which an attacker attempts a single common password against many user accounts. This approach reduces the chance of triggering account lockout policies while identifying valid credentials.

How It Works

  1. The attacker collects a list of valid usernames.
  2. A common password is tested against all accounts.
  3. Authentication attempts are spread over time to avoid lockouts.
  4. Valid credentials are used to gain access.

Requirements

  • A list of valid usernames.
  • Access to an authentication service such as Kerberos, LDAP, SMB, RDP, VPN, or O365.
  • Weak or commonly used passwords.

Failure Conditions

  • Strong, unique passwords.
  • Multi-Factor Authentication (MFA).
  • Account lockout or Conditional Access policies.
  • Invalid usernames.

Usage Scenarios

  • Initial access against Active Directory or cloud identities.
  • Validating usernames discovered during reconnaissance.
  • Environments with weak password policies or password reuse.

Important Notes

Always check the account lockout policy before testing.
