Pass-the-Hash (PtH)

Pass-the-Hash (PtH) is an authentication technique that allows an adversary to authenticate to remote systems using an NTLM password hash instead of the user's plaintext password. By abusing the NTLM authentication protocol, attackers can move laterally, execute remote commands, and access network resources without cracking the hash.

Requirements

  • A valid NTLM hash of the target account.
  • The target must support NTLM authentication.
  • Valid account with remote access privileges.
  • Administrative privileges on the target for remote execution methods (e.g., PsExec, WMI, SMB, WinRM).

Advantages

No need to crack the password. Fast and effective for lateral movement. Uses legitimate NTLM authentication.

Disadvantages

Only works against NTLM, not pure Kerberos authentication. Authentication attempts and remote execution can be logged and detected.
