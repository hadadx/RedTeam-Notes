AS-REP Roasting

AS-REP Roasting is a Kerberos attack that targets Active Directory accounts with Kerberos Pre-Authentication disabled. It allows an attacker to obtain encrypted authentication data from the KDC and perform offline password cracking to recover the user's password.

How It Works

  1. The attacker identifies an account with Pre-Authentication disabled.
  2. An AS-REQ is sent without pre-authentication data.
  3. The KDC returns an AS-REP encrypted with a key derived from the user's password.
  4. The response is cracked offline to recover the plaintext password.

Requirements

  • Valid Active Directory username.
  • Kerberos Pre-Authentication disabled on the target account.
  • Network access to the Domain Controller over Kerberos.

Failure Conditions

  • Kerberos Pre-Authentication is enabled.
  • The account is invalid or disabled.
  • The password is too strong to crack.

Usage Scenarios

  • Target accounts with Kerberos Pre-Authentication disabled.
  • If GenericWrite or GenericAll permissions are available, an attacker can temporarily disable Kerberos Pre-Authentication, request the AS-REP hash, and restore the original account settings afterward.
