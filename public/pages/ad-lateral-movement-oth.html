Overpass-the-Hash (OtH)

Overpass-the-Hash (also called "pass-the-key") is a lateral movement technique that upgrades an NTLM password hash into a full Kerberos Ticket Granting Ticket (TGT). Instead of reusing the hash the traditional NTLM way, the attacker feeds it into the Kerberos authentication process to request legitimate Kerberos tickets. These tickets can then be used to obtain service tickets (TGS) and gain remote code execution as the target user, while avoiding NTLM authentication over the network.

Requirements

  • A valid NTLM hash of the target account (typically dumped from memory with a tool like Mimikatz).
  • The target domain must support Kerberos authentication.
  • A valid domain account (the one whose hash was obtained).
  • Ability to run the process in the target user's context (e.g., sekurlsa::pth from Mimikatz).
  • For remote execution, appropriate privileges on the target and a Kerberos-aware tool (e.g., PsExec, WMI, WinRM).

Advantages

No need to crack the password. Converts a hash into legitimate Kerberos tickets, so activity blends in with normal domain traffic. Stealthier than traditional Pass-the-Hash because it avoids NTLM authentication. Enables use of any Kerberos-based tooling once the TGT is generated.

Disadvantages

Requires generating a TGT (usually by triggering a domain action such as net use), adding an extra step over plain PtH. Ticket requests are still recorded by the Key Distribution Center (KDC) and can be detected. Relies on a Kerberos-enabled environment. Note: whoami will still show the original user, since it only reads the process token, not imported Kerberos tickets.
