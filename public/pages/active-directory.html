Active Directory

Active Directory (AD) is Microsoft's centralized directory service for managing identities, authentication, authorization, and resources within a Windows domain. It enables administrators to centrally manage users, computers, groups, policies, and permissions while providing secure access to network resources.

Objectives

  • Centralize identity and access management.
  • Authenticate users and computers within the domain.
  • Authorize access to resources based on permissions.
  • Enforce security policies across the organization.
  • Simplify administration of large enterprise environments.

Core Components

Domain

A logical administrative boundary that contains users, computers, groups, and other directory objects sharing a common database and security policies.

Domain Controller (DC)

A server that hosts Active Directory, stores the directory database, authenticates users, enforces security policies, and replicates data with other Domain Controllers.

Forest

The highest-level Active Directory structure. A forest contains one or more domains that share a common schema, configuration, and Global Catalog.

Tree

A collection of one or more domains connected through a hierarchical namespace and automatic trust relationships.

Organizational Unit (OU)

A logical container used to organize directory objects and apply administrative delegation and Group Policies.

Directory Objects

  • Users
  • Computers
  • Groups
  • Printers
  • Shared folders
  • Service Accounts
  • Organizational Units

Authentication

Active Directory primarily uses Kerberos for authentication and falls back to NTLM when Kerberos cannot be used.

Authentication is performed through the Key Distribution Center (KDC) running on the Domain Controller.

Authorization

After authentication, Active Directory determines which resources a user can access based on:

  • Security Groups
  • Access Control Lists (ACLs)
  • Access Control Entries (ACEs)
  • User privileges and permissions

Group Policy (GPO)

Group Policy allows administrators to centrally configure security settings, software deployment, user configurations, scripts, and operating system policies across users and computers within the domain.

DNS Integration

Active Directory relies heavily on DNS for locating Domain Controllers, resolving services (SRV records), and enabling clients to discover authentication and directory services.

Replication

Changes made on one Domain Controller are replicated to other Domain Controllers to maintain a consistent directory database across the domain or forest.

Benefits

  • Centralized administration
  • Single Sign-On (SSO)
  • Scalable identity management
  • Policy-based security enforcement
  • Simplified resource management
  • High availability through replication
  • Fine-grained access control
